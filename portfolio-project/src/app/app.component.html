Personal Projects

The modular server framework allows different Python modules to be written that define how to process the data from the client in different layers: the socket, encryption, and application layers. While this is a simple model, it greatly decreases the amount of code that needs to be rewritten for different scenarios. I found that when I wanted to write custom server software, I would often need to rewrite a large portion of the code. With this framework, I can use the same socket and encryption modules between multiple projects and only change the applicaion modules to fit the specific need of the application I'm working on.


I used this framework to create an imaging automation tool using the Python server and a C# client to run custom-made PowerShell scripts on client machines. In an imaging scenario, this could include app installations, device configurations, and virtually any other function that can be completed through a PowerShell script.


The process starts by the client reporting to the server with a password to verify that it's a valid client. The two then exchange public keys and begin data transmission. The server sends PowerShell scripts and runtime configuration settings and arguments to the client, which are then parsed and executed. The client and server both use hybrid encryption (RSA with AES) to secure the traffic between them, ensuring that no third-parties, even with full access to the conversations, could decrypt the messages.
